📚 Library ERP – Installation Guide for Windows, macOS & Linux

This guide covers installation on Windows (XAMPP/WAMP), macOS (MAMP or native), and provides quick references for other Linux distributions.

2. Installation on Windows (XAMPP / WAMP)

Recommended: Use XAMPP (Apache + PHP + SQLite support).

2.1 Install XAMPP

  • Download XAMPP for Windows from https://www.apachefriends.org/ (PHP 8.2 or 8.3 version).
  • Run the installer – accept defaults (C:\xampp).
  • Launch XAMPP Control Panel and start Apache.
  • Ensure PHP extensions are enabled: open C:\xampp\php\php.ini and uncomment (remove semicolon) the following lines: 
    extension=sqlite3
extension=gd
extension=zip
extension=curl
extension=mbstring
extension=fileinfo
extension=intl
;extension=zip
  • Restart Apache from XAMPP Control Panel.

2.2 Place the Library ERP Files

  • Download and UNZIP the Software package file .
    •
  • Copy the library-system folder to C:\xampp\htdocs\.
  • Rename if desired (e.g., library).

2.3 Set Permissions (Windows)

  • Right‑click the uploads, backups, and config folders → Properties → Security → Give Modify/Write permission to Everyone or the web server user (IUSR or SYSTEM).
  • Alternatively, run the following in Command Prompt as Administrator (adjust path): 
    icacls "C:\xampp\htdocs\library-system\uploads" /grant Everyone:F /T
icacls "C:\xampp\htdocs\library-system\backups" /grant Everyone:F /T
icacls "C:\xampp\htdocs\library-system\config" /grant Everyone:F /T

2.4 Access the System

Open browser and go to http://localhost/library-system/. The database will be created automatically.

OPAC= http://localhost/library-system/opac

VISITOR LOG= http://localhost/library-system/visitor

Default Username= admin Password= admin123
💡 If you get a "Forbidden" error, check that .htaccess files are allowed. In XAMPP, edit C:\xampp\apache\conf\extra\httpd-xampp.conf and ensure AllowOverride All is set for the htdocs directory.

2.5 Optional: Use WAMP (alternative)

  • Install WAMP from wampserver.com.
  • Enable required PHP extensions via the WAMP tray icon → PHP → Extensions.
  • Place the library folder inside C:\wamp64\www\.
  • Access at http://localhost/library-system/.http://localhost/library-system/opac http://localhost/library-system/visitor

3. Installation on macOS

Two methods: using MAMP (simplest) or native Apache + PHP.

3.1 Using MAMP (Recommended for Beginners)

  • Download MAMP from https://www.mamp.info/ (free version).
  • Install and launch MAMP. Set PHP version to 8.x in preferences.
  • Copy the library-system folder to /Applications/MAMP/htdocs/.
  • Set permissions (Terminal): 
    cd /Applications/MAMP/htdocs
chmod -R 755 library-system/
chmod -R 777 library-system/uploads library-system/backups library-system/config
  • Start MAMP servers and access http://localhost:8888/library-system/ (or http://localhost/library-system/ if ports are standard).

3.2 Using Native Apache & PHP (Advanced)

macOS includes Apache and PHP, but PHP 8.3 may need to be installed via Homebrew.

  • Install Homebrew if not already: /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  • Install PHP 8.3: brew install php@8.3
  • Enable Apache module: sudo brew services start php@8.3 (or link the module).
  • Place library folder in /Library/WebServer/Documents/: 
    sudo cp -r library-system /Library/WebServer/Documents/
  • Set permissions: 
    cd /Library/WebServer/Documents
sudo chown -R _www:_www library-system/
sudo chmod -R 755 library-system/
sudo chmod -R 777 library-system/uploads library-system/backups library-system/config
  • Edit Apache config (/etc/apache2/httpd.conf) to enable mod_rewrite and allow Override.
  • Restart Apache: sudo apachectl restart
  • Access at http://localhost/library-system/.

    4.2 Step‑by‑Step Installation on Ubuntu

    4.2.1 Update System & Install Apache

    sudo apt update && sudo apt upgrade -y
sudo apt install -y gedit
sudo apt install -y apache2
sudo systemctl enable apache2 && sudo systemctl start apache2

    4.2.2 Install PHP 8.3 & Required Extensions

    sudo apt install -y php8.3 libapache2-mod-php8.3 php8.3-sqlite3 php8.3-gd php8.3-zip \
php8.3-mbstring php8.3-curl php8.3-xml php8.3-intl

    Verify with php -v (should show PHP 8.3.x).

    4.2.3 Enable Apache Modules & Upload Files

    sudo a2enmod rewrite
sudo systemctl restart apache2

Option B: If you have a ZIP file
    cd /var/www/html

Option C: Using Git (if you have a repository)
    cd /var/www/html

    4.2.4 Set Correct Permissions

    cd /var/www/html
sudo chown -R www-data:www-data /var/www/html/library-system/
sudo chmod -R 755 /var/www/html/library-system/
sudo chmod -R 777 /var/www/html/library-system/uploads/
sudo chmod -R 777 /var/www/html/library-system/backups/
sudo chmod -R 777 /var/www/html/library-system/config/     # Important for SQLite DB creation

    4.2.5 Configure Virtual Host (Recommended)

    sudo gedit /etc/apache2/sites-available/library.conf

    Paste the following:

    <VirtualHost *:80>
    ServerName library.local
    DocumentRoot /var/www/html/library-system
    <Directory /var/www/html/library-system>
        Options Indexes FollowSymLinks
        AllowOverride All
        Require all granted
    </Directory>
    ErrorLog ${APACHE_LOG_DIR}/library-error.log
    CustomLog ${APACHE_LOG_DIR}/library-access.log combined
</VirtualHost>
    sudo a2ensite library.conf
sudo a2dissite 000-default.conf
sudo systemctl reload apache2

sudo gedit /etc/hosts
Add 127.0.0.1   library.local

    Optional: add 127.0.0.1 library.local to /etc/hosts for local testing.

    5. Installation on Other Linux Distributions (CentOS/RHEL/Fedora)

    The process is similar to Ubuntu. Use your package manager (yum, dnf) to install Apache, PHP 8.x, and required extensions.

    5.1 Example for CentOS 9 / RHEL 9 / Fedora

    # Install EPEL and Remi repositories
sudo dnf install epel-release
sudo dnf install https://rpms.remirepo.net/enterprise/remi-release-9.rpm
sudo dnf module reset php
sudo dnf module install php:remi-8.3

# Install Apache and PHP extensions
sudo dnf install httpd php php-sqlite3 php-gd php-zip php-mbstring php-curl php-xml php-intl

# Enable and start Apache
sudo systemctl enable httpd
sudo systemctl start httpd

# Place files in /var/www/html/
sudo cp -r library-system /var/www/html/
sudo chown -R apache:apache /var/www/html/library-system/
sudo chmod -R 755 /var/www/html/library-system/
sudo chmod -R 777 /var/www/html/library-system/{uploads,backups,config}

# Allow .htaccess overrides
sudo nano /etc/httpd/conf/httpd.conf   # set AllowOverride All for the document root

sudo systemctl restart httpd

    Access at http://your-server-ip/library-system/.

    8. Cron Jobs (Automated Tasks)

    Edit crontab (sudo crontab -e) and add the following lines:

    # Daily tasks (overdue reminders, fines, email queue)
0 2 * * * /usr/bin/php /var/www/html/library-system/cron/daily_tasks.php >> /var/log/library-cron.log 2>&1

# Process email queue every 5 minutes (if using SMTP)
*/5 * * * * /usr/bin/php /var/www/html/library-system/cron/process_email_queue.php >> /var/log/library-email.log 2>&1

# Weekly backup rotation & Google Drive sync (optional)
0 3 * * 1 /usr/bin/php /var/www/html/library-system/cron/weekly_tasks.php >> /var/log/library-cron.log 2>&1

    Adjust paths according to your installation.

    9. PHP Tuning (for large uploads)

    sudo gedit /etc/php/8.3/apache2/php.ini

    Set:

    upload_max_filesize = 100M
post_max_size = 100M
max_execution_time = 600
memory_limit = 512M

    Restart Apache: sudo systemctl restart apache2

    10. Security Recommendations

    • Change all default passwords (admin, staff1).
    • Enable HTTPS with Let's Encrypt: sudo certbot --apache -d yourdomain.com
    • Ensure .htaccess files are active (block config/, backups/, vendor/).
    • Regularly update system: sudo apt update && sudo apt upgrade -y

    11. Troubleshooting Quick Reference

    IssueSolution
    White screen / 500 errorCheck Apache error logs: sudo tail -f /var/log/apache2/error.log. Verify PHP extensions.
    Database not createdEnsure config/ is writable (chmod 777 config/).
    Upload failsCheck uploads/ permissions and PHP upload_max_filesize.
    Email not sendingTest SMTP in Settings → Notifications. Check queue with php cron/process_email_queue.php.
    PDF export blankInstall dompdf: composer require dompdf/dompdf or use browser print.
