2. Windows – XAMPP

2.1 Install XAMPP

  • Download XAMPP for Windows from apachefriends.org (PHP 8.2 or 8.3 version).
  • Run the installer – accept defaults (C:\xampp).
  • Launch XAMPP Control Panel and start Apache.

2.2 Enable Required PHP Extensions

  • Open C:\xampp\php\php.ini in Notepad.
  • Uncomment (remove the leading ;) these lines: 
    extension=sqlite3
extension=gd
extension=zip
extension=curl
extension=mbstring
extension=fileinfo
extension=intl
  • Save and restart Apache from the XAMPP Control Panel.

2.3 Deploy the Application

  • Extract the downloaded Library-Management-System-main.zip.
  • Inside it, find and extract library-system.zip → you get the library-system folder.
  • Copy the library-system folder to C:\xampp\htdocs\.

2.4 Set Permissions

Right‑click the uploads, backups, and config folders inside library-system → Properties → Security → give Modify/Write permission to Everyone or the web server user.

Alternatively, run as Administrator:

icacls "C:\xampp\htdocs\library-system\uploads" /grant Everyone:F /T
icacls "C:\xampp\htdocs\library-system\backups" /grant Everyone:F /T
icacls "C:\xampp\htdocs\library-system\config" /grant Everyone:F /T
⚠️ Refresh to create database: If the database is not created automatically on the first visit, simply refresh the page 3–4 times. The system may need a few requests to initialise all tables. Ensure the config/ folder is writable (permissions already set above).

2.5 Access the System

Open browser: http://localhost/library-system/

  • Default login: admin / admin123
  • OPAC: http://localhost/library-system/opac
  • Visitor Log: http://localhost/library-system/visitor

3. macOS

4.2 Set Correct Permissions

$ cd /var/www/html
$ sudo chown -R www-data:www-data /var/www/html/library-system/
$ sudo chmod -R 755 /var/www/html/library-system/
$ sudo chmod -R 777 /var/www/html/library-system/uploads/           # If fail, repeat after installation
$ sudo chmod -R 777 /var/www/html/library-system/backups/           # If fail, repeat after installation
$ sudo chmod -R 777 /var/www/html/library-system/config/            # Important for SQLite DB creation

4.3 Configure Virtual Host (Recommended)

# Create a new virtual host configuration
$ sudo gedit /etc/apache2/sites-available/library.conf

Paste the following:

<VirtualHost *:80>
    ServerName library.local
    DocumentRoot /var/www/html/library-system
    <Directory /var/www/html/library-system>
        Options Indexes FollowSymLinks
        AllowOverride All
        Require all granted
    </Directory>
    ErrorLog ${APACHE_LOG_DIR}/library-error.log
    CustomLog ${APACHE_LOG_DIR}/library-access.log combined
</VirtualHost>
# Enable the site and reload Apache
$ sudo a2ensite library.conf
$ sudo a2dissite 000-default.conf
$ sudo systemctl reload apache2

# Add to /etc/hosts for local access
$ echo "127.0.0.1 library.local" | sudo tee -a /etc/hosts

6. Performance & Error‑Free Configuration

6.1 PHP Settings (Recommended)

Edit your php.ini file to match these values for optimal performance:

memory_limit = 512M
upload_max_filesize = 100M
post_max_size = 100M
max_execution_time = 600
max_input_time = 600
date.timezone = Asia/Kolkata   # adjust to your region

# Enable OPCache for faster execution
opcache.enable = 1
opcache.memory_consumption = 128
opcache.max_accelerated_files = 10000
opcache.revalidate_freq = 0       # disable for development, set 60 in production

# Enable error logging (useful for debugging)
error_reporting = E_ALL & ~E_DEPRECATED & ~E_STRICT
display_errors = Off
log_errors = On
error_log = /var/log/php_errors.log

After changes, restart your web server.

6.2 Enable SQLite WAL Mode (Already Set)

The system automatically enables PRAGMA journal_mode=WAL for better concurrency and performance – no action required.

6.3 Configure .htaccess for Security

Ensure the following .htaccess file is present in the root of your library-system folder to block direct access to sensitive directories:

<IfModule mod_rewrite.c>
    RewriteEngine On

    # Block access to config, backups, vendor, etc.
    RewriteRule ^(config|backups|vendor|logs|temp|cache)/ - [F,L]

    # Prevent directory listing
    Options -Indexes

    # Redirect to index.php for clean URLs (if needed)
    # RewriteCond %{REQUEST_FILENAME} !-f
    # RewriteCond %{REQUEST_FILENAME} !-d
    # RewriteRule ^(.*)$ index.php [QSA,L]
</IfModule>

7. Optional – Composer for Excel & PDF Reports

If you need advanced reporting (Excel, PDF), install Composer and the required packages:

$ cd /var/www/html/library-system
$ sudo apt install -y composer   # or brew install composer on macOS
$ composer require phpoffice/phpspreadsheet
$ composer require dompdf/dompdf
If Composer is not available, the system falls back to HTML/print‑based PDFs.

8. Scheduled Tasks (Cron)

Set up these cron jobs to automate overdue notifications, email processing, and backup rotation:

$ sudo crontab -e
# Daily tasks (overdue reminders, fine calculation)
0 2 * * * /usr/bin/php /var/www/html/library-system/cron/daily_tasks.php >> /var/log/library-cron.log 2>&1

# Process email queue every 5 minutes
*/5 * * * * /usr/bin/php /var/www/html/library-system/cron/process_email_queue.php >> /var/log/library-email.log 2>&1

# Weekly backup rotation (keeps last 10 backups)
0 3 * * 1 /usr/bin/php /var/www/html/library-system/cron/weekly_tasks.php >> /var/log/library-cron.log 2>&1
Adjust paths if your installation is in a different directory.

9. Security Hardening

  • Change default passwords immediately after login (admin, staff1).
  • Enable HTTPS using Let's Encrypt: sudo certbot --apache -d yourdomain.com (Ubuntu) or equivalent.
  • Restrict access to config/, backups/, uploads/ via .htaccess (included).
  • Enable fail2ban to block brute‑force attacks on login.
  • Regular updates: sudo apt update && sudo apt upgrade -y.
  • Pull latest code from GitHub periodically.

10. Backup & Restore

10.1 Create a Backup

  • Log in as admin → System Settings → Backup.
  • Click "Create Backup" – a .zip file will be saved in the backups/ folder.
  • Backups include the SQLite database and all uploaded files (covers, eBooks).

10.2 Restore from a Backup

  • In the same Backup section, select a previous backup and click "Restore".
  • The system will replace the current database and files with the backup version.
⚠️ Always create a fresh backup before performing a restore or update.

11. Troubleshooting Quick Reference

IssueSolution
ZipArchive not available Enable extension=zip in php.ini and restart the web server.
White screen / 500 error Check Apache error logs: sudo tail -f /var/log/apache2/error.log. Verify all PHP extensions are installed.
Database not created Refresh the page 3–4 times; if still fails, ensure config/ is writable (run permission commands again).
Upload fails Check uploads/ permissions and PHP upload_max_filesize.
Email not sending Test SMTP in Settings → Notifications. Check the queue with php cron/process_email_queue.php.
PDF export blank Install dompdf via Composer, or use the browser's print function as fallback.
Slow performance Enable OPCache, increase memory limit, and ensure SQLite WAL mode is active (it is by default).
