📋 Manual Installation – Quick Start (Linux / Ubuntu / CentOS)

Use these commands to clone or download the repository directly to your server's web root.

📁 Important – Folder Structure: The ZIP file extracts as Library-Management-System-main which contains a folder named library-system (the actual application). The commands below handle this automatically. 
# Option 1: Clone with Git (recommended for easy updates)
git clone https://github.com/sameemonm1-hue/library-system.git

# Option 2: Download and extract the ZIP
wget https://github.com/sameemonm1-hue/library-system/archive/main.zip
unzip main.zip
mv library-system-main/library-system ./   # move the inner folder to current directory
rm -rf library-system-main main.zip        # clean up

# Finally, move the 'library-system' folder to your web root (e.g., /var/www/html/)
sudo mv library-system /var/www/html/
💡 Pro Tip: Using git clone lets you pull future updates with a single git pull origin main command.
✅ After moving the folder, continue with the permissions and virtual host setup in the platform‑specific sections below.

2. Installation on Windows (XAMPP)

Recommended: Use XAMPP (Apache + PHP + SQLite support).

2.1 Download Library ERP from GitHub

⬇️ Step 1: Download the software from GitHub:

  • Option A: Click the "Download ZIP" button in the download box above.
  • Option B: Visit https://github.com/sameemonm1-hue/library-system and click the green "Code" button → "Download ZIP".
  • Option C: Use Git: git clone https://github.com/sameemonm1-hue/library-system.git
💡 Always download from the official GitHub repository to ensure you have the latest version and security updates.

2.2 Install XAMPP

  • Download XAMPP for Windows from https://www.apachefriends.org/ (PHP 8.2 or 8.3 version).
  • Run the installer – accept defaults (C:\xampp).
  • Launch XAMPP Control Panel and start Apache.

2.3 Enable ZipArchive Extension (Critical for Backups)

⚠️ Important: If you see the error "ZipArchive not available. Backups will be saved as folders", follow these steps:

  1. Open C:\xampp\php\php.ini in Notepad or any text editor.
  2. Press Ctrl + F and search for ;extension=zip
  3. Remove the semicolon (;) to uncomment it: 
    extension=zip
  4. If you find ;extension=php_zip.dll instead, change it to: 
    extension=php_zip.dll
  5. If the line doesn't exist, add it at the bottom of the file.
  6. Save the file and restart Apache from XAMPP Control Panel.
✅ Verify: Create a phpinfo() file and search for "zip" – you should see a section confirming ZipArchive is enabled.

2.4 Enable All Required PHP Extensions

In the same php.ini file, uncomment (remove semicolon) these lines:

extension=sqlite3
extension=gd
extension=zip          ← Critical for compressed backups
extension=curl
extension=mbstring
extension=fileinfo
extension=intl

Restart Apache after making changes.

2.5 Place the Library ERP Files

  • Extract the downloaded ZIP file from GitHub.
  • Inside the extracted Library-Management-System-main folder, locate the library-system folder (this contains the application).
  • Copy the library-system folder to C:\xampp\htdocs\.
  • Rename if desired (e.g., library).

2.6 Set Permissions (Windows)

  • Right‑click the uploads, backups, and config folders (inside the copied library-system folder) → Properties → Security → Give Modify/Write permission to Everyone or the web server user (IUSR or SYSTEM).
  • Alternatively, run the following in Command Prompt as Administrator (adjust path): 
    icacls "C:\xampp\htdocs\library-system\uploads" /grant Everyone:F /T
icacls "C:\xampp\htdocs\library-system\backups" /grant Everyone:F /T
icacls "C:\xampp\htdocs\library-system\config" /grant Everyone:F /T

2.7 Access the System

Open browser and go to http://localhost/library-system/. The database will be created automatically.

  • OPAC: http://localhost/library-system/opac
  • Visitor Log: http://localhost/library-system/visitor

Default Login: Username: admin | Password: admin123

💡 If you get a "Forbidden" error, check that .htaccess files are allowed. In XAMPP, edit C:\xampp\apache\conf\extra\httpd-xampp.conf and ensure AllowOverride All is set for the htdocs directory.

2.8 Alternative: WAMP

  • Install WAMP from wampserver.com.
  • Enable required PHP extensions via the WAMP tray icon → PHP → Extensions (specifically enable php_zip).
  • Place the library-system folder inside C:\wamp64\www\.
  • Access at http://localhost/library-system/.

3. Installation on macOS

Two methods: using MAMP (simplest) or native Apache + PHP.

3.1 Download from GitHub

⬇️ Download the package:

  • Visit https://github.com/sameemonm1-hue/library-system
  • Click the green "Code" button → "Download ZIP"
  • Or use Git: git clone https://github.com/sameemonm1-hue/library-system.git
After extraction, locate the library-system folder inside Library-Management-System-main – that is the application folder.

3.2 Using MAMP (Recommended for Beginners)

  • Download MAMP from https://www.mamp.info/ (free version).
  • Install and launch MAMP. Set PHP version to 8.x in preferences.
  • Copy the extracted library-system folder to /Applications/MAMP/htdocs/.
  • Set permissions (Terminal): 
    cd /Applications/MAMP/htdocs
chmod -R 755 library-system/
chmod -R 777 library-system/uploads library-system/backups library-system/config
  • Enable ZipArchive in MAMP: Open /Applications/MAMP/bin/php/php8.x/conf/php.ini and uncomment extension=zip.
  • Start MAMP servers and access http://localhost:8888/library-system/.

3.3 Using Native Apache & PHP (Advanced)

  • Install Homebrew: /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  • Install PHP 8.3: brew install php@8.3
  • Enable ZipArchive: brew install php@8.3 --with-zip or ensure it's enabled in /usr/local/etc/php/8.3/php.ini.
  • Enable Apache module: sudo brew services start php@8.3
  • Place library-system folder in /Library/WebServer/Documents/: 
    sudo cp -r library-system /Library/WebServer/Documents/
  • Set permissions: 
    cd /Library/WebServer/Documents
sudo chown -R _www:_www library-system/
sudo chmod -R 755 library-system/
sudo chmod -R 777 library-system/uploads library-system/backups library-system/config
  • Edit Apache config (/etc/apache2/httpd.conf) to enable mod_rewrite and allow Override.
  • Restart Apache: sudo apachectl restart
  • Access at http://localhost/library-system/.

4.2 Step‑by‑Step Installation

Update System & Install Apache

sudo apt update && sudo apt upgrade -y
sudo apt install -y apache2 git unzip
sudo systemctl enable apache2 && sudo systemctl start apache2

Install PHP 8.3 & Required Extensions (Including ZipArchive)

sudo apt install -y php8.3 libapache2-mod-php8.3 php8.3-sqlite3 php8.3-gd \
php8.3-zip php8.3-mbstring php8.3-curl php8.3-xml php8.3-intl

Verify PHP version: php -v (should show PHP 8.3.x).

Verify ZipArchive is enabled: php -m | grep zip (should return "zip").

Deploy the Files (using the manual steps above)

Refer to the Manual Installation box at the top of this guide for the clone/download commands.

After obtaining the library-system folder, move it to the web root (eg /home):

sudo mv library-system /var/www/html/

Enable Apache Modules

sudo a2enmod rewrite
sudo systemctl restart apache2

Set Correct Permissions

cd /var/www/html
sudo chown -R www-data:www-data /var/www/html/library-system/
sudo chmod -R 755 /var/www/html/library-system/
sudo chmod -R 777 /var/www/html/library-system/uploads/           # If fail repeat after installtion
sudo chmod -R 777 /var/www/html/library-system/backups/           # If fail repeat after installtion
sudo chmod -R 777 /var/www/html/library-system/config/            # Important for SQLite DB creation

Configure Virtual Host (Recommended)

sudo gedit /etc/apache2/sites-available/library.conf

Paste the following:

<VirtualHost *:80>
    ServerName library.local
    DocumentRoot /var/www/html/library-system
    <Directory /var/www/html/library-system>
        Options Indexes FollowSymLinks
        AllowOverride All
        Require all granted
    </Directory>
    ErrorLog ${APACHE_LOG_DIR}/library-error.log
    CustomLog ${APACHE_LOG_DIR}/library-access.log combined
</VirtualHost>
sudo a2ensite library.conf
sudo a2dissite 000-default.conf
sudo systemctl reload apache2

# Add to /etc/hosts
echo "127.0.0.1 library.local" | sudo tee -a /etc/hosts

Access the System- (Refresh or Restart)

http://library.local/              # Main system
http://library.local/opac          # OPAC
http://library.local/visitor       # Visitor Log

Default: admin / admin123

Use the Manual Installation commands at the top of this guide to clone or download the repository, then move the library-system folder to /var/www/html/.

6. Fix: "ZipArchive not available" Error

If you see this error:

"ZipArchive not available. Backups will be saved as folders. Enable extension=zip in php.ini for compressed backups."

Solution by Platform:

Windows (XAMPP):

1. Open C:\xampp\php\php.ini
2. Find ;extension=zip
3. Remove semicolon → extension=zip
4. Save and restart Apache

Windows (WAMP):

1. Click WAMP tray icon → PHP → Extensions
2. Enable php_zip
3. Restart WAMP services

macOS (MAMP):

1. Open /Applications/MAMP/bin/php/php8.x/conf/php.ini
2. Uncomment extension=zip
3. Restart MAMP servers

Ubuntu/Debian:

sudo apt install php8.3-zip
sudo systemctl restart apache2

CentOS/RHEL/Fedora:

sudo dnf install php-zip
sudo systemctl restart httpd
✅ After fixing: Go to Admin → System Settings → Backup and try creating a backup. It should now create a .zip file instead of a folder.

7. First Run & Database Setup

The system will automatically create the SQLite database and all tables on first access.

✅ No manual SQL commands needed.

Default admin credentials:

  • Username: admin  |  Password: admin123
  • Staff test account: staff1 / staff123

After login, immediately change the admin password via Admin → System Settings → Change Admin Password.

⚠️ REFRESH – repeat three or more times for creating DB

If the database is not created automatically on the first visit, simply refresh the page 3–4 times. The system may need a few requests to initialise all tables.

If the database still doesn't appear, ensure the config/ folder is writable (already set in permissions above). Additionally, run these commands to set full permissions on the uploads/ and backups/ folders:

cd /var/www/html
sudo chmod -R 777 /var/www/html/library-system/uploads/
sudo chmod -R 777 /var/www/html/library-system/backups/
✅ After running the commands, refresh the page again – the database should be created.

12. Troubleshooting Quick Reference

IssueSolution
ZipArchive not availableEnable extension=zip in php.ini and restart web server (see Section 6).
White screen / 500 errorCheck Apache error logs: sudo tail -f /var/log/apache2/error.log. Verify PHP extensions.
Database not createdRefresh page 3–4 times; if still fails, run the permission commands in Section 7.
Upload failsCheck uploads/ permissions and PHP upload_max_filesize.

14. Backup System

Once ZipArchive is enabled, the system can create compressed backups:

  • Go to Admin → System Settings → Backup
  • Click "Create Backup"
  • Backups will be saved as .zip files in the backups/ folder
  • You can download or restore from existing backups
💡 The backup includes: SQLite database, all uploaded files (book covers, eBooks), and system configuration.
